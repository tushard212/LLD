giObserver Design Pattern

Why it was needed
- Problem: When one object (the subject) changes state, many other objects (observers) need to be notified and updated. Without a pattern, the subject would need direct knowledge of concrete observer classes, causing tight coupling and poor maintainability.
- Need: Decouple the subject from its observers so observers can be added/removed at runtime with minimal changes.

What problem it solves
- Implements a one-to-many dependency between objects so that when one object changes state, all its dependents are notified and updated automatically.

How it solves the problem (concept)
- Define an `Observer` interface with an `update()` method.
- Define a `Subject` that keeps a list of `Observer` references and exposes `attach()`, `detach()`, and `notify()` methods.
- Concrete observers implement `update()` and register themselves with the subject. When subject state changes it calls `notify()` which calls `update()` on each observer.

CLASS FLOW / ARCHITECTURE:
=====================================

        ┌──────────────────┐
        │  <<Interface>>   │
        │    Observer      │
        │──────────────────│
        │ + update(state)  │
        └────────┬─────────┘
                 △
                 │
        ┌────────┴────────┬──────────┐
        │                 │          │
   ┌──────────────┐  ┌──────────────┐
   │ ConcreteObs1 │  │ ConcreteObs2 │
   ├──────────────┤  ├──────────────┤
   │ + update()   │  │ + update()   │
   └──────────────┘  └──────────────┘
   (Concrete Observers)

        ┌──────────────────┐
        │    Subject       │
        │──────────────────│
        │ - observers[]    │
        │ - state          │
        ├──────────────────┤
        │ + attach(obs)    │ ←─── Observers
        │ + detach(obs)    │      register
        │ + notify()       │      here
        │ + setState()     │
        └──────────────────┘
                 │
                 │ when state changes,
                 │ notify() calls
                 ▼
        Observer::update()
        on all observers

Flow:
1. Observer calls subject.attach(this)
2. Subject stores observer reference
3. When subject state changes → setState()
4. setState() calls notify()
5. notify() calls update() on each observer

C++ Example (simple, self-contained)

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
#include <memory>

// Observer interface
struct Observer {
    virtual ~Observer() = default;
    virtual void update(int newState) = 0;
};

// Subject base
class Subject {
public:
    void attach(std::shared_ptr<Observer> obs) { observers.push_back(obs); }
    void detach(std::shared_ptr<Observer> obs) {
        observers.erase(std::remove(observers.begin(), observers.end(), obs), observers.end());
    }
protected:
    void notify(int state) {
        for (auto &w : observers) {
            if (auto obs = w.lock()) {
                obs->update(state);
            }
        }
    }
private:
    std::vector<std::weak_ptr<Observer>> observers;
};

// Concrete subject
class ConcreteSubject : public Subject {
public:
    void setState(int s) {
        state = s;
        notify(state);
    }
    int getState() const { return state; }
private:
    int state{0};
};

// Concrete observer
class ConcreteObserver : public Observer, public std::enable_shared_from_this<ConcreteObserver> {
public:
    ConcreteObserver(const std::string &n, std::shared_ptr<ConcreteSubject> s)
        : name(n), subject(s) {}
    void subscribe() {
        subject->attach(shared_from_this());
    }
    void unsubscribe() {
        subject->detach(shared_from_this());
    }
    void update(int newState) override {
        std::cout << name << " received state: " << newState << '\n';
    }
private:
    std::string name;
    std::shared_ptr<ConcreteSubject> subject;
};

int main() {
    auto subject = std::make_shared<ConcreteSubject>();

    auto o1 = std::make_shared<ConcreteObserver>("ObserverA", subject);
    auto o2 = std::make_shared<ConcreteObserver>("ObserverB", subject);

    o1->subscribe();
    o2->subscribe();

    subject->setState(10); // both observers get notified

    o2->unsubscribe();

    subject->setState(20); // only ObserverA gets notified

    return 0;
}
```

Notes about the example
- Uses `std::weak_ptr` in the subject to avoid strong reference cycles and allow observers to be destroyed safely.
- `enable_shared_from_this` allows an observer to get a `shared_ptr` to itself when attaching.

Advantages
- Loose coupling: subject and observers interact through interfaces, not concrete classes.
- Dynamic: observers can be added/removed at runtime.
- Broadcast communication: one-to-many updates are simple to implement.

Disadvantages / Caveats
- Notification order and timing can be unclear; observers may depend on update order.
- Can cause performance issues if many observers are notified frequently.
- Potential for memory leaks or dangling pointers if ownership is not managed correctly—use smart pointers (or careful manual management) in C++.
- Harder to reason about in multi-threaded contexts — requires synchronization.

When to use
- Use when you need a clean one-to-many dependency and want to decouple the subject from its observers (e.g., UI event systems, publish/subscribe, model-view updates).


================================================================================
YouTube Example — IChannel / ISubscriber
================================================================================

- `ISubscriber` interface  → only `update()` method
- `IChannel` interface     → `subscribe()`, `unsubscribe()`, `notify()` methods
- `YouTubeChannel` (concrete) implements `IChannel`, holds a list of `ISubscriber*`
- `Subscriber` (concrete) implements `ISubscriber`, holds a reference to `YouTubeChannel`

```cpp
#include <iostream>
#include <vector>
#include <string>
#include <algorithm>

// ──────────────────────────────────────
// Interface: ISubscriber  (Observer role)
// ──────────────────────────────────────
class ISubscriber {
public:
    virtual ~ISubscriber() = default;
    virtual void update(const std::string& videoTitle) = 0;   // only method
};

// ──────────────────────────────────────
// Interface: IChannel  (Subject role)
// ──────────────────────────────────────
class IChannel {
public:
    virtual ~IChannel() = default;
    virtual void subscribe(ISubscriber* sub)   = 0;
    virtual void unsubscribe(ISubscriber* sub) = 0;
    virtual void notify(const std::string& videoTitle) = 0;
};

// ──────────────────────────────────────
// Concrete Subject: YouTubeChannel
//   implements IChannel
// ──────────────────────────────────────
class YouTubeChannel : public IChannel {
public:
    YouTubeChannel(const std::string& name) : channelName(name) {}

    void subscribe(ISubscriber* sub) override {
        subscribers.push_back(sub);
        std::cout << "[" << channelName << "] New subscriber added.\n";
    }

    void unsubscribe(ISubscriber* sub) override {
        subscribers.erase(
            std::remove(subscribers.begin(), subscribers.end(), sub),
            subscribers.end()
        );
        std::cout << "[" << channelName << "] Subscriber removed.\n";
    }

    void notify(const std::string& videoTitle) override {
        for (auto* sub : subscribers) {
            sub->update(videoTitle);
        }
    }

    // channel-specific behaviour
    void uploadVideo(const std::string& title) {
        std::cout << "\n[" << channelName << "] Uploaded: \"" << title << "\"\n";
        notify(title);          // notify all subscribers
    }

private:
    std::string channelName;
    std::vector<ISubscriber*> subscribers;
};

// ──────────────────────────────────────
// Concrete Observer: Subscriber
//   implements ISubscriber
//   has-a relationship with YouTubeChannel
// ──────────────────────────────────────
class Subscriber : public ISubscriber {
public:
    Subscriber(const std::string& name, YouTubeChannel& ch)
        : subscriberName(name), channel(ch) {}

    // convenience helpers that delegate to the channel
    void subscribeTo()     { channel.subscribe(this); }
    void unsubscribeFrom() { channel.unsubscribe(this); }

    // ISubscriber — only method
    void update(const std::string& videoTitle) override {
        std::cout << "  >> " << subscriberName
                  << " got notified: \"" << videoTitle << "\"\n";
    }

private:
    std::string subscriberName;
    YouTubeChannel& channel;       // relationship with the channel
};

// ──────────────────────────────────────
// main — demo
// ──────────────────────────────────────
int main() {
    YouTubeChannel techChannel("TechWithTim");

    Subscriber alice("Alice", techChannel);
    Subscriber bob("Bob",     techChannel);
    Subscriber carol("Carol", techChannel);

    alice.subscribeTo();
    bob.subscribeTo();
    carol.subscribeTo();

    techChannel.uploadVideo("Observer Pattern Explained");
    // All three get notified

    bob.unsubscribeFrom();

    techChannel.uploadVideo("Factory Pattern Explained");
    // Only Alice and Carol get notified

    return 0;
}
```

Expected output:
    [TechWithTim] New subscriber added.
    [TechWithTim] New subscriber added.
    [TechWithTim] New subscriber added.

    [TechWithTim] Uploaded: "Observer Pattern Explained"
      >> Alice got notified: "Observer Pattern Explained"
      >> Bob got notified: "Observer Pattern Explained"
      >> Carol got notified: "Observer Pattern Explained"

    [TechWithTim] Subscriber removed.

    [TechWithTim] Uploaded: "Factory Pattern Explained"
      >> Alice got notified: "Factory Pattern Explained"
      >> Carol got notified: "Factory Pattern Explained"
